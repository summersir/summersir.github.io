本地运行方法

  • 命令行运行npm install
  • 模块下载成功后，运行 node server 启动服务器
  • 打开浏览器访问127.0.0.1:3000（源码里设置的就是3000端口号）

准备工作

本文示例环境为Mac，Windows也就是Node的安装与命令行稍有不同，程序的实现基本与平台无关。

Node相关

  • 你需要在本机安装Node.js
  • 多少需要一点Node.js的基础知识，如果还未曾了解过Node.js，这里有一篇不错的入门教程

然后我们就可以开始创建一个简单的服务器了。

mime

我们要用js实现发送消息到服务器，再由服务器处理后吐出数据显示在HTML上，所以要使用mime模块来响应css、js文件的请求。

安装mime

  • 在命令行中输入 npm install mime --save 回车进行安装
  • 在server.js中通过 var mime = require('mime') 将其引入到项目中

socket

安装socket
官网教程写了详细的方法

Node.js中使用socket的一个包。使用它可以很方便地建立服务器到客户端的sockets连接，发送事件与接收特定事件。
同样通过npm进行安装 npm install socket.io 。安装后在node_modules文件夹下新生成了一个socket.io文件夹。

  • 在命令行中输入 npm install socket.io --save 回车进行安装

在socket.io文件夹我们可以找到一个socket.io.js文件。将它引入到HTML页面，这样我们就可以在前端使用socket.io与服务器进行通信了。

  • index.html链接服务器

同时服务器端的server.js里跟使用express一样，也要通过require(‘socket.io’)将其引入到项目中，这样就可以在服务器端使用socket.io了。

在html界面记得要绑定好可以链接的地址：var socket = io('http://127.0.0.1:3000');在后面可以会解释原因。

  • server.js绑定服务器

用法

使用socket.io，其前后端句法是一致的，即通过socket.emit()来激发一个事件，通过socket.on()来侦听和处理对应事件。这两个事件通过传递的参数进行通信。
详细代码我就不再这里写了请看源码的HTML和server.js

具体流程

客户端发送消息

在客户端点击发送按钮，获取input的输入值value，并且将数据通过socket.emit发送给服务器

// js发送消息代码

服务端接收消息

服务端接通过socket.on收到消息后处理消息

注意：客户端发送数据时的字段，要与服务端接收数据时的字段保持一致，例如：

客户端发送new字段数据：socket.emit('news', { info: message });
服务端接收new字段数据：socket.on('news', function (data) {...});

服务端返还数据

服务端讲接收到的数据处理好，再通过io.sockets.emit发送到客户端（由html获取数据）

客户端接收返还数据

客户端通过socket.on接收到服务端返还的数据后通过相应的处理让他呈现在客户端（HTML）上显示出来

注意：客户端发送数据时的字段，要与服务端接收数据时的字段保持一致，例如：

客户端发送message字段数据：io.sockets.emit('message',{msg:arr});
服务端接收message字段数据：socket.on('message', function (data) {...});

这样我们的简易的聊天室就基本搭建好了

启动服务器

终端命令：node server
在前文中提到过，在html界面记得要绑定好可以链接的地址：var socket = io('http://127.0.0.1:3000');
众所周知127.0.0.1只是本机的IP，如果想实现线上链接那么这个地址就要换成你联网后的IP地址，Mac是打开网络偏好设置就可看到，Windows我有些忘记了，可自行百度查找。
在设置好IP后，在浏览器的地址栏输入你设置的IP+:3000端口号后就可进入聊天服务器了，也可让你的朋友登录这个地址也一起来聊天发消息。

后记

这个简易的聊天记录还是十分简单的，没有设置昵称，没有表情，没有图片，不能清屏等等，还有好多好多的不足，但是这通过这个我们基本上可以了解Node.js是如何创建并启动服务器的，之后还是会用到很多框架，进行完善的。
